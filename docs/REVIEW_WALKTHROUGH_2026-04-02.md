## 一、核心問題診斷

### 🔴 P0-1：三個 Section 模組邊界模糊（T2 + S1）

**現狀**：

```
Section ① 你的現況
├── 公司選擇 Chip（功能 B 遺留）
├── 制度對照表 8 維度（功能 B 遺留）
├── PARAM_SCHEMA 輸入欄位（功能 A）
│   ├── A. 現狀盤點
│   ├── B. 轉職風險
│   └── C. 保經優勢
└── 全部混在同一個 Section ① 裡
```

**問題**：

- T2：「我看到 Section ① 叫『你的現況』，但裡面既有公司選擇、又有制度對照表、又有輸入欄位——到底哪個是『我的現況』？」
- S1：功能 A（試算引擎）和功能 B（制度對照）在 v3.0 合併後，UI 上沒有做視覺分隔，使用者無法區分「資訊展示區」和「互動輸入區」
- S1：制度對照表和輸入欄位之間沒有過渡，使用者不知道「我需要往下滑才有輸入框」

**T2 走查實錄**：
> 「我打開頁面，看到一堆公司 Chip，以為是選我要比較的公司。接著看到對照表，覺得『好，資訊到了』。但滾下去發現還有輸入欄位？我已經看完對照表了啊，為什麼還要我輸入東西？」

### 🔴 P0-2：觀念溝通話術被藏在 Accordion 裡，面談時找不到（T1 + P1）

**現狀**：

```
Section ③ 進階探討（預設收合）
├── 🔬 三情境比較（Accordion — 預設收合）
└── 🧠 面談備忘（Accordion — 預設收合）
    ├── 第一層思維（需點擊展開）
    ├── 第二層思維（需點擊展開）
    └── 第三層思維（需點擊展開）
```

**問題**：

- T1：「面談備忘是我最需要的東西，但它被藏在 Section ③ 最底下，還要點兩次才能看到內容（先開 Accordion、再點卡片）。面談時我需要偷看一眼話術提示，不可能在對方面前找半天。」
- P1：L5 教案「初次面談兩步驟用法」明確說面談只需要「制度對照 30 秒 + 數字對照 30 秒」，三層思維是**跟進用的**，但目前 UI 沒有區分「初次面談模式」和「深度跟進模式」
- T1：「v2 的 Tab 4 雖然不完美，但至少話術在一個獨立 Tab 裡，我知道它在哪。現在合併後反而更難找了。」

**缺失話術盤點**：

| 話術類型 | 教案位置 | 網站狀態 | 說明 |
|---------|---------|---------|------|
| 三層思維腳本 | L5 | ✅ 有（5身份×3層=15組） | 但藏在 Accordion |
| 初次面談用法 | L5 新增 | ❌ 缺 | 「制度→數字→回本」三步流程 |
| 異議處理話術 | L4（15項） | ❌ 缺 | 面談中最常被問的反駁 |
| 破冰開場話術 | L1 | ❌ 缺 | 「你一天花多少比例在真正想做的事上？」 |
| LAER 回應框架 | L4 | ❌ 缺 | Listen→Acknowledge→Explore→Respond |
| 保經 vs 保經差異化 | L4 #12 | ❌ 缺 | 六大精神差異 |
| 50歲轉換回應 | L4 #14 | ❌ 缺 | 時間框架翻轉 |

### 🟡 P1-1：Section ② 數字說話 —— KPI 四格缺脈絡引導（T1）

**問題**：

- T1：「KPI 四格直接丟出數字，但面談時我需要一句引導語：『你現在的制度，五年能累積 X 萬。如果同樣的努力放在保經制度下……』——這個過渡語完全沒有。」
- KPI 區只有數字，沒有「觀念溝通」的框架
- 「收入超越點」是好指標，但旁邊沒有話術提示告訴顧問「怎麼用這個數字說服對方」

### 🟡 P1-2：Section ① 制度對照表缺「說故事」引導（T1 + P1）

**問題**：

- 對照表有 8 個維度的「資料」，但沒有「怎麼講」的提示
- T1：「面談時我不是逐行唸表格，我需要知道：對方是保險同業時，哪三個維度最有說服力？銀行理專時，重點又是哪幾個？」
- P1：L5 教案已定義每個身份的「重點維度」，但網站沒有對應的視覺提示（如高亮或排序）

**身份別建議重點維度**：

| 身份 | 核心打擊點（前3） | 教案來源 |
|------|-----------------|---------|
| 保險同業 | 產品選擇、收入天花板、合約制度 | L4 #1-#3 |
| 銀行理專 | 首年佣金率（4-5倍差距）、客戶所有權、底薪 vs 終生佣酬 | L4 #6, L5 |
| 壽險主管 | 組織發展支援、收入天花板、合約制度（傳承） | L4 #7-#8 |
| 醫療通路 | 首年佣金率、產品選擇、培訓體系 | L5 醫療專區 |
| 新人 | 合約制度（從第一天開始）、培訓體系、產品選擇 | L1 + L5 |

## 二、模組架構重整建議（S1 + P1 聯合提案）

### 現狀問題：一頁三段式 → 段落邊界不清

```
現狀 UI 流程：
[身份選擇] → [公司Chip + 對照表 + 輸入欄位（全混在Section①）]
           → [KPI + 對比圖（Section②）]
           → [情境 + 話術（Section③，藏起來）]
```

### 建議：重新定義四個功能區塊

```
建議 UI 流程（面談說服動線）：

🟦 區塊 A — 「你的制度」（30秒）
   身份選擇 + 公司選擇 + 制度對照表
   → 目的：讓對方「認出自己的處境」
   → 話術提示：「你現在的制度長這樣…」

🟨 區塊 B — 「數字怎麼說」（30秒）
   核心輸入（3-4個）+ KPI 四格 + 時間軸
   → 目的：用數字製造「衝擊感」
   → 話術提示：「同樣的努力，換一個平台…」
   → 進階輸入折疊在「調整假設」裡

🟪 區塊 C — 「萬一不順利呢」（選用）
   三情境比較 + 逐年對比 + 十年圖
   → 目的：回應「你太樂觀了」的質疑
   → 話術提示：「即使用最保守的假設…」

🟩 區塊 D — 「面談話術庫」（顧問專用，對方看不到）
   ├── 🏷️ 初次面談三步驟（L5）
   ├── 🏷️ 三層思維腳本（5身份×3層）
   ├── 🏷️ 常見異議速查（L4 前5項）
   └── 🏷️ LAER 回應框架
   → 可加密或需手勢才能展開（面談時不讓對方看到）
```

**核心改變**：
1. 把「制度對照」和「輸入欄位」拆開 → 對照是「定性」、輸入是「定量」
2. 話術從「藏在 Accordion」升格為獨立區塊
3. 每個區塊有明確的面談用途標籤（30秒/選用/顧問專用）
4. 輸入欄位精簡：always 欄位移到區塊 B，advanced 折疊

## 五、結論與行動建議

### 當前可用度評分（六角色平均）

| 面向 | 評分 | 說明 |
|------|------|------|
| 面談實用性 | 6.5/10 | 數字引擎強大，但面談動線被 UI 架構拖累 |
| 話術完整性 | 4/10 | 三層思維有，但初次面談話術、異議處理、引導語全缺 |
| 模組清晰度 | 5/10 | v3.0 合併後比 v2.0 Tab 好，但三段式邊界不清 |
| 教案銜接 | 5.5/10 | L5 教的操作流程與網站實際 UI 不完全對應 |
| 合規 | 8.5/10 | 基本面通過，細節有改善空間 |
| 技術品質 | 8/10 | PARAM_SCHEMA 架構優秀，程式碼結構清晰 |

### 建議執行順序

**Phase 1（本次可做）**：
1. 🔴 修復理專黃金交叉計算
2. 🔴 重新定義 Section 標題與視覺分隔
3. 🟡 在 Section ③ 新增「初次面談三步驟」（從 Accordion 提取為獨立區塊）

5. 整合 L4 異議處理前 5 項速查

8. 面談模式 vs 課堂模式切換
9. 深度跟進話術整合
10. 手機端 UX 優化

---

**審查結論**：MDRT-BPS v3.0.2 的計算引擎和 PARAM_SCHEMA 架構是堅實的技術底座，但 UI 的資訊架構和面談話術整合是最大缺口。最關鍵的改善不是加新功能，而是**重新劃清模組邊界 + 把已經寫好的教案話術搬進來**。
