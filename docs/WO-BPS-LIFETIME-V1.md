# WO-BPS-LIFETIME-V1：三段式遠景收入模型 + 沉沒成本 + 風險因子

> **版本**：v1.0 ｜ **建立日期**：2026-04-10
> **狀態**：規格確認
> **專案**：mdrt-bps（獨立於保策通）

---

## 核心問題

現行 `compute5yr` 只看 5 年，無法回答增員對象最在意的問題：
> 「我在這間公司待了 10 年，現在跳出去，長期來看到底划不划算？」

但 25 年單線投射不可信（假設崩塌 + 合規紅線 + 結論必然偏向保經）。

**解法：三段式漸進遠景 — 越遠越模糊，搭配沉沒成本清單 + 風險因子面板。**

---

## 功能設計三問

### Q1：誰要用？用在何處？
增員主管面對「猶豫型」資深業務員（35-50 歲，年資 10+）。
場景：面談第二輪 — 對方已有基礎認知，想看「長期怎麼樣」。
**不取代現行 5 年模型**，作為 Tab B 的進階區塊（Section ③ 進階內）。

### Q2：目的與功能適配程度？
讓對方「自己看清楚」三件事：
1. 時間拉長後兩邊的趨勢方向
2. 留在原公司的隱性損失（沉沒成本）
3. 「做到退休」的真實機率

現有 `compute5yr` + `calc10YearIncome` 可延伸，不需重寫。

### Q3：讓產品更好用，還是更複雜？
更好用。資深業務員最常說的反對理由就是「我已經待這麼久了」，這個功能直接回應。
但必須做成**可選展開**（折疊），不影響核心 5 年流程。

## SDD（系統設計）

### 三段式模型

| 階段 | 年限 | 計算方式 | 呈現 | 資料來源 |
|------|------|---------|------|---------|
| 精確期 | 1-5 年 | `compute5yr`（現行） | 逐年雙線 + KPI 四格 | 用戶輸入參數 |
| 延伸期 | 6-10 年 | `computeExtended` 三情境 | 三線區間帶 | 基準 ± 成長率偏移 |
| 趨勢期 | 11-退休 | `computeTrend` 上下界 | 扇形色帶（漸淡） | 歷史趨勢 + 衰減模型 |

### 三情境定義

| 情境 | FYP 成長 | 佣金率變動 | 脫落率 | 說明 |
|------|---------|----------|-------|------|
| 保守 | 0% | -1%/5年 | +5% | 佣金率下調、保單脫落增加 |
| 基準 | 現行值 | 不變 | 現行值 | 延續 5 年假設 |
| 樂觀 | +3%/年 | 不變 | -3% | 業績成長、脫落率降低 |

### 新增輸入參數

| 參數 | key | 預設 | 範圍 | 放置位置 |
|------|-----|------|------|---------|
| 目前年齡 | `currentAge` | 35 | 22-60 | 進階區塊頂部 |
| 預計退休年齡 | `retireAge` | 60 | 45-70 | 進階區塊頂部 |

### 沉沒成本勾選面板

| 項目 | key | 說明 | 預設勾選 |
|------|-----|------|---------|
| 退職金/養老金 | `sunk_pension` | 大型壽險公司內部制度 | ✓ |
| 團體保險額度 | `sunk_groupIns` | 離開即歸零 | ✓ |
| 續期佣金 | `sunk_renewal` | 離開後 1-5 年停發 | ✓ |
| 組織利益 | `sunk_orgBonus` | 團隊瓦解損失 | ☐ |
| 年終/季獎金 | `sunk_yearEnd` | 按比例回吐 | ☐ |

每項可選填金額（選填），僅加總顯示「離開成本參考值」，**不參與投射計算**。

### 風險因子面板

| 指標 | 資料來源 | 呈現 |
|------|---------|------|
| 13 月定著率 | 壽險公會年報（公開） | 存活曲線圖 |
| 做到退休的機率 | 定著率反推（假設等比衰減） | 百分比 + 色碼 |
| 近 5 年佣金率趨勢 | 業界報導 | 文字說明 |

### UI 結構

```
Section ③ 進階（現有）
└─ <details> 遠景分析（點擊展開）
   ├─ 年齡輸入列（currentAge / retireAge）
   ├─ 「剩餘 N 年工作年限」心理錨定
   ├─ 三段式圖表（精確→延伸→趨勢）
   ├─ 沉沒成本勾選面板
   ├─ 風險因子面板
   └─ 免責聲明（固定底部）
```

### 免責聲明（強制）

- 圖表旁：「超過 5 年之投射為趨勢性估算，可靠性隨年限遞減。」
- 底部完整版：「本試算僅為數學模型推演，不構成收入保證或承諾。實際收入受個人業績、公司政策、法規變動、市場環境等因素影響。」

## BDD（驗收條件）

### Scenario 1：年齡輸入 → 三段式圖表

```gherkin
Given 用戶輸入 currentAge=35, retireAge=60
When 展開遠景分析
Then 顯示「剩餘 25 年工作年限」
And 精確期（Y1-5）顯示雙線逐年數字
And 延伸期（Y6-10）顯示三線情境帶（樂觀/基準/保守）
And 趨勢期（Y11-25）顯示漸淡扇形色帶，無具體數字
And 圖表底部顯示免責聲明
```

### Scenario 2：沉沒成本勾選

```gherkin
Given 用戶勾選「退職金」填入 500000、「團險額度」填入 200000
When 查看沉沒成本區塊
Then 顯示「離開成本參考值：NT$ 700,000」
And 不影響收入投射圖表的數字
```

### Scenario 3：風險因子

```gherkin
Given 用戶展開遠景分析
When 查看風險因子面板
Then 顯示「25 年做到退休的機率：約 3-5%」（基於定著率計算）
And 顯示存活曲線
```

### Scenario 4：異業身份不顯示

```gherkin
Given CID = 'realtor' 或 'auto'
When 查看 Section ③
Then 遠景分析區塊不顯示（異業無傳統公司沉沒成本）
```

## Error Path

| 情境 | 處理 |
|------|------|
| retireAge ≤ currentAge | 紅字提示「退休年齡需大於目前年齡」 |
| 年齡差 > 40 | 提示「工作年限超過 40 年，數據僅供參考」 |
| 沉沒成本全未勾選 | 面板顯示「無選擇項目」，不影響其他功能 |
| compute5yr 返回 NaN | 隱藏三段式圖表，顯示「請先完成基本參數設定」 |
