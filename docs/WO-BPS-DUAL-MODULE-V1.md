## 功能設計三問

**Q1：誰要用？用在何處？**
公勝增員顧問，在面談桌上（手機/平板）對準增員對象展示。兩種場景：
- 場景 A：對方是其他行業（銀行理專、醫療、壽險主管），尚未決定要做保險 → 需要痛點反思
- 場景 B：對方已有意願做保險（同業或新人），在比較不同公司制度 → 需要收入試算

**Q2：目的與功能適配程度？**
目前 v3.0 把兩種場景混在同一頁（身份切換器 + 公司 chip），造成操作衝突。拆成兩個獨立模組，各自有清晰的使用流程。現有 PARAM_SCHEMA、compute5yr、COMPANY_DB 全部複用。

**Q3：讓產品更好用，還是更複雜？**
更好用。顧問進頁面第一步就是選「我要做什麼」，不再困惑。拿掉會被抱怨——因為現在混在一起正在被抱怨。

## 模組 A：為什麼做保經？

### 目標
讓準增員對象「自己意識到現職痛點」，而非被推銷。顧問只需要遞出手機，讓對方自己操作。

### 資料來源（教案 L1-L5 萃取）

| 維度 | 教案來源 | 內容 |
|------|---------|------|
| 收入天花板 | L1 投影片 3-5 | 壽險 19-25% vs 保經 50%+，10年複利差距 |
| 客戶歸屬權 | L1 投影片 6-7、L2 | 離職=客戶歸零 vs 終身跟隨 |
| 工作自主權 | L2、L4 | 早會/考核/KPI 束縛 vs 自由安排 |
| 職涯成長 | L1、L3 | 固定晉升階梯 vs 六級無天花板 |
| 退休保障 | L1 投影片 7-8 | 退休歸零 vs 世代傳承 |

### UI 元件

#### A-1：身份選擇（現有 5 persona，保留）
- 保險同業 / 銀行理專 / 壽險主管 / 醫療通路 / 新人
- 選擇後整個 Tab A 內容切換

#### A-2：痛點雷達圖（新增）
- 5 軸雷達圖：收入天花板、客戶歸屬、工作自由、職涯成長、退休保障
- 每軸 1-10 分，依身份預設（可手動調整）
- 左側：現職得分（紅色，偏低）｜ 右側：保經得分（綠色，偏高）
- 資料驅動：`PERSONA_PAIN_POINTS` 靜態物件

預設分數範例（保險同業）：
```
                現職   保經
收入天花板       4      9
客戶歸屬權       3      10
工作自主權       5      8
職涯成長         4      9
退休保障         3      9
```

#### A-3：生活節奏對照卡（新增）
- 一天的時間軸（07:00-22:00），左右對照
- 左：現職典型一天（有早會、有打卡、有 KPI 壓力）
- 右：保經典型一天（自由安排、客戶導向）
- 依身份不同切換內容（銀行理專有分行輪值、醫療有門診時段）
- 資料來源：L2 對象分類的身份描述 + L4 異議處理的情境描述

#### A-4：收入天花板視覺化（改造現有 income-chart）
- 保留現有 10 年累計收入差距圖
- 但資料改用身份的「典型值」，不需手動輸入
- 銀行理專：底薪 80 萬 + 佣金 8% → vs 保經 40%
- 壽險同業：佣金 20% → vs 保經 50%
- 加入「沉沒成本標記」：第 N 年開始超越

#### A-5：觀念溝通話術（改造現有 SCRIPTS）
- 保留現有三層思維腳本
- 但從折疊式改為**卡片輪播**（更適合面談遞手機）
- 每張卡片一個觀點，左右滑動
- 末頁 CTA：「想看具體數字？→ 切到收入試算」

#### A-6：決策觸發 CTA
- 按鈕：「幫你算算看 →」
- 點擊 → 自動切換到 Tab B，並帶入身份對應的預設公司

## 模組 B：制度收入試算

### 目標
在對方已有初步意願後，用精確數字呈現「轉職的具體損益」。

### 變更（相對 v3.0.3）

| 項目 | v3.0.3 | v4.0 |
|------|--------|------|
| 身份切換器 | 在 Section ① 內 | 移除（由 Tab A 帶入，或 Tab B 頂部小選單） |
| 公司 chip | 壽險 + 保經雙軌 | 只有壽險/自訂（保經固定公勝） |
| 比較表右欄 | 動態 selectedBrokerId | 固定公勝 |
| 入口 | 直接進入 | 從 Tab A CTA 過渡，或直接點 Tab B |

### 保留不動的部分
- PARAM_SCHEMA 全部欄位定義
- compute5yr 計算引擎
- COMPANY_DB 全部資料
- 制度對照表（COMPARISON_DIMS）
- KPI 四格 + 5 年逐年對比
- 三情境比較
- 面談備忘（三層思維腳本）
- 10 年收入差距圖

### Tab B 頂部小選單
- 一行文字：「對方身份：🛡️ 保險同業 ▾」（下拉切換，預設由 Tab A 帶入）
- 下方：公司 chip（壽險 + 自訂，無保經）
- 其餘佈局與現有 Section ①②③ 相同

## BDD 驗收條件

### Tab 切換
```gherkin
Given 使用者進入 MDRT-BPS 頁面
When 頁面載入完成
Then 顯示兩個 Tab：「為什麼做保經？」「制度收入試算」
And 預設顯示 Tab A

Given 使用者在 Tab A
When 點擊「制度收入試算」Tab
Then Tab B 內容顯示，Tab A 隱藏
And URL hash 更新為 #calculator
```

### Tab A — 痛點反思
```gherkin
Given 使用者在 Tab A
When 選擇「銀行理專」身份
Then 雷達圖顯示銀行理專的痛點分數
And 生活節奏對照卡顯示銀行的典型一天
And 觀念溝通話術切換為銀行理專版本
And 收入天花板圖用銀行理專典型值渲染

Given 使用者在 Tab A 瀏覽完畢
When 點擊「幫你算算看 →」CTA
Then 自動切換到 Tab B
And Tab B 身份預設為「銀行理專」
And Tab B 公司預設為「國泰」（或上次選擇）
```

### Tab B — 收入試算
```gherkin
Given 使用者在 Tab B
When 點擊公司 chip「南山」
Then 傳統端參數填入南山預設值
And 比較表左欄顯示「南山」，右欄固定「公勝」
And KPI、圖表隨即更新

Given 使用者在 Tab B
When 修改任一參數
Then 所有計算結果即時更新（≤100ms）
And 修改欄位顯示「已手動調整」標記
```

## Error Path 清單

| 情境 | UI 處理 |
|------|---------|
| Tab A 雷達圖渲染失敗 | 顯示純文字痛點列表（降級） |
| Tab B 公司 defaults 缺少當前身份 | 降級到 insurance defaults |
| compute5yr 回傳 NaN | KPI 顯示「請檢查輸入」 |
| 手機端 Tab 切換卡頓 | Tab 切換不做動畫，直接 display toggle |
